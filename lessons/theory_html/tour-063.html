Урок 63: Методы и косвенность указателей

Сравнивая предыдущие две программы, вы можете заметить, что функции с аргументом-указателем должны принимать указатель:

вар v вершина ScaleFunc(v, 5) // Ошибка компиляции! ScaleFunc(&v, 5) // ОК

в то время как методы с приемниками указателей принимают либо значение, либо указатель в качестве приемник, когда они вызываются:

вар v вершина v.Scale(5) // ОК п := &v p.Scale(10) // ОК

Для оператора `v.Scale(5)`, хотя `v` является значением, а не указателем, метод с приемником указателя вызывается автоматически. То есть для удобства Go интерпретирует оператор `v.Scale(5)` как `(&v).Scale(5)`, поскольку метод Scale` имеет приемник указателя.

Пример для подростков: Представь, что объясняешь тему другу на перемене: если можешь просто рассказать, ты понял материал.

Немного юмора: Маленькие шаги в Go работают лучше всего: запустил, проверил, улучшил — и без паники.
