Урок 75: Ошибки

Программы Go выражают состояние ошибки с помощью значений error.

Тип error — это встроенный интерфейс, аналогичный fmt.Stringer:

тип ошибки интерфейса { Строка ошибки() }

(Как и в случае с fmt.Stringer, пакет fmt ищет интерфейс error при печать значений.)

Функции часто возвращают значение ошибки, а вызывающий код должен обрабатывать ошибки. проверяя, равна ли ошибка `nil`.

я, ошибка:= strconv.Atoi("42") если ошибка != ноль { fmt.Printf("не удалось преобразовать число: %v\n", ошибка) возвращение } fmt.Println("Преобразованное целое число:", i)

Нулевая ошибка означает успех; ненулевая `ошибка` означает неудачу.

Пример для подростков: Представь, что объясняешь тему другу на перемене: если можешь просто рассказать, ты понял материал.

Немного юмора: Маленькие шаги в Go работают лучше всего: запустил, проверил, улучшил — и без паники.
