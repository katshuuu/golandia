Редактирование теории курса (быстрый режим)
============================================

Где править текст
-----------------
1) Одним файлом на урок: положите сюда HTML с именем <id_урока>.html
   Например: tour-002.html подменит theory_html для урока tour-002 из module_*.json.

2) Введение курса (как на дашборде): файл _intro.html подменит intro_html из course_manifest.json.

Правило: если файл существует и не пустой — он побеждает значение из JSON (удобнее длинные статьи
править в отдельном файле с подсветкой HTML/Emmet, чем экранировать кавычки в JSON).

Что использовать в разметке
---------------------------
Иерархия: h2 → h3 → h4; списки ul/ol; блоки blockquote для цитат.
Акценты:
  <strong>важно</strong>  — автоматически выделяется в теории (жирнее + акцентный оттенок).
  <em>логический акцент</em> — курсив и чуть мягче по цвету.

Размер:
  Абзац крупнее: <p class="theory-text--lg">...</p>
  Мельче (второстепенное): <p class="theory-text--sm theory-muted">...</p>

Блок «обрати внимание» с рамкой:
  <div class="theory-callout"><p><strong>Важно:</strong> краткая мысль.</p></div>

Ключевая формулировка в строке:
  <span class="theory-key-point">одна главная мысль</span>

Код: <pre><code>...</code></pre> в теории уже оформляется как блок «окна».

Оглавление и метаданные по-прежнему в course_manifest.json (название курса, модули, список id уроков,
заголовки в сайдбаре). Полный текст теории — здесь или в JSON, на выбор.

После сохранения .html перезапустите бэкенд (он читает файлы при старте).
