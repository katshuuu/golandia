Урок 68: Интерфейсы с nil-значениями

Если конкретное значение внутри самого интерфейса равно нулю, метод будет вызываться с нулевым получателем.

В некоторых языках это может вызвать исключение нулевого указателя. но в Go принято писать методы, которые корректно обрабатывают вызовы. с нулевым получателем (как в случае с методом «M» в этом примере.)

Обратите внимание, что значение интерфейса, которое содержит нулевое конкретное значение, само по себе не является нулевым.

Пример для подростков: Интерфейс — как универсальный порт зарядки: важно, чтобы устройство поддерживало нужный разъём.

Немного юмора: Go не требует слова implements: если методы совпали, тип уже в команде.
