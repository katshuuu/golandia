Урок 66: Неявная реализация интерфейсов

Тип реализует интерфейс, реализуя его методы. Нет ни явного заявления о намерениях, ни ключевого слова «реализует».

Неявные интерфейсы отделяют определение интерфейса от его описания. реализацию, которая затем могла бы появиться в любом пакете без предварительной договоренности.

Пример для подростков: Интерфейс — как универсальный порт зарядки: важно, чтобы устройство поддерживало нужный разъём.

Немного юмора: Go не требует слова implements: если методы совпали, тип уже в команде.
