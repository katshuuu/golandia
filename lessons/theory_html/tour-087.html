Урок 87: Операторы range и close

Отправитель может «закрыть» канал, чтобы указать, что значения больше не будут отправляться. Получатели могут проверить, был ли закрыт канал, назначив второй параметр выражению приема: после

v, ок := <-ch

`ok` имеет значение `false`, если больше нет значений для приема и канал закрыт.

Цикл `for`i`:=`range`c` получает значения из канала повторно, пока он не будет закрыт.

*Примечание:* Закрывать канал должен только отправитель, а не получатель. Отправка по закрытому каналу вызовет панику.

*Еще одно*примечание:* Каналы не похожи на файлы; обычно вам не нужно их закрывать. Закрытие необходимо только тогда, когда получателю необходимо сообщить, что больше не поступают значения, например, для завершения цикла `range`.

Пример для подростков: Представь, что объясняешь тему другу на перемене: если можешь просто рассказать, ты понял материал.

Немного юмора: Маленькие шаги в Go работают лучше всего: запустил, проверил, улучшил — и без паники.
