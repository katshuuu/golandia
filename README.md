# LLM-репетитор по Go

Интерактивная веб-платформа для изучения языка программирования **Go**, где путь от первого `fmt.Println` до горутин ощущался как последовательный и достижимый квест. Умный **AI-помощник** отвечает на вопросы в контексте урока и вашего кода, а встроенная **песочница** позволяет запускать и проверять решения без установки компилятора на компьютер.

---

## Опции платформы:

| Направление | Описание |
|-------------|----------|
| **Структурированное обучение** | Модули и порядок уроков следуют логике Tour of Go: от базового синтаксиса до обобщений и конкурентности. |
| **Теория + практика в одном окне** | Вкладки «Теория» и «Задание», демокод из уроков, редактор с сохранением черновика в браузере. |
| **Мгновенная обратная связь** | Запуск кода в песочнице и автоматическая проверка по правилам (точный вывод, regex, список подстрок, комбинированные проверки). |
| **Персонализация** | Цель обучения на платформе, профиль с прогрессом и формулировкой цели, восстановление пароля. |
| **Поддержка на каждом шаге** | AI-помощник получает контекст урока, ваш код и последний вывод — ответы ближе к репетитору, чем к «голому» чату. |

---

## Преимущества приложения:

1. **Не нужно настраивать Go локально для старта** — основной сценарий «написал → запустил → получил результат» работает в браузере.

2. **Один источник правды для курса** — содержание и задания уроков задаются JSON-файлами (`lessons/`), что упрощает правки текста и проверок без пересборки бизнес-логики.

3. **Быстрое восприятие интерфейса** — боковая навигация по модулям, выдвижная панель уроков, визуальный прогреш по задачам, отдельные экраны профиля и достижений.

4. **Надёжная работа с прогрессом** — интеграция с Supabase для задач и решений; локальный кэш прогресса даёт более плавную загрузку после перезагрузки страницы.

5. **Безопасная и понятная аутентификация** — вход и регистрация через Supabase Auth, возможность восстановления доступа к аккаунту.

6. **Расширяемость** — типы проверок заданий (`output`, `contains`, `regex`, `forbidden`, `combined`) позволяют точно воспроизводить постановки упражнений вроде официальных tour-задач.

## Геймификация и мотивация

Платформа не сводится к «прочитал страницу» — закреплены элементы, которые поддерживают вовлечённость и ощущение роста.

- **Уровень и роль** — на экране достижений отображается персонаж и роль (например, «Стажёр»), плюс блоки «Мой уровень» с темами в виде карточек-«слотов»: от базового синтаксиса до горутин. Это визуально показывает путь Junior → уверенный Go-разработчик.

- **Счётчики и проценты** — «решено заданий», процент прохождения курса, прогресс по модулям в сайдбаре: пользователь всегда видит, сколько уже сделано и куда двигаться дальше.

- **Мини-проекты по разделам** — в манифесте курса у модулей заложены мини-проекты с шагами и итоговым результатом («раздел пройден, задания без ошибок»), что задаёт финальную цель блока.

- **Итоговый челлендж** — в `course_manifest.json` описан **финальный проект тура** с целями, подсказками и своей проверкой (`tour complete`), чтобы закрыть обучение осознанным итогом.

- **Мгновенные награды за код** — при успешной проверке задания показываются позитивные уведомления и статус «Решено»; код в редакторе сохраняется между визитами к уроку.

- **Личная цель** — при первом входе (и в профиле) пользователь формулирует цель на платформе; это привязывает обучение к реальному намерению («найти работу», «сдать экзамен», «перейти на Go в команде»).

- **Подсказки к заданиям** — опциональные текстовые подсказки через данные задач из БД смягчают фрустрацию и уменьшают отток на сложных уроках.

## Основной стек

- **React 18** + **TypeScript**
- **Vite** — сборка и dev-сервер
- **Tailwind CSS** — стили
- **Supabase** — аутентификация, таблицы `tasks`, `solutions`, edge-функции для AI
- Локальный курс: **JSON** (`lessons/module_*.json`, `course_manifest.json`)
- Встроенная **песочница** для ограниченного выполнения Go в браузере

## Быстрый старт

### Требования

- Node.js 18+ (рекомендуется LTS)
- npm

### Установка и запуск

```bash
cd golandia
npm install
npm run dev
```

По умолчанию в `package.json` dev-сервер слушает **127.0.0.1** и порт из скрипта `dev` (см. актуальный `package.json`). Откройте в браузере адрес, который выведет Vite (например `http://127.0.0.1:5174/`).

### Сборка production

```bash
npm run build
npm run preview   # локальный просмотр сборки
```

### Переменные окружения

Создайте файл `.env` в корне проекта (значения возьмите из панели Supabase):

```
VITE_SUPABASE_URL=https://<project>.supabase.co
VITE_SUPABASE_ANON_KEY=<anon_key>
```

Без них вход и сохранение прогресса в облаке работать не будут; локальные части интерфейса могут быть доступны ограниченно.

## Структура курса в репозитории

```
lessons/
  course_manifest.json   # оглавление, модули, финальный проект, мини-проекты
  module_01.json … module_07.json   # уроки: theory_html, demo_code, task (проверки)
```

Редактирование контента: правьте соответствующие JSON и перезапустите dev или пересоберите проект.
